# PRD：運動紀錄功能

**Status:** Draft
**Owner:** Simon
**Last Updated:** 2026-04-19
**Target Release:** —

---

## 背景

Pace 是一款 iOS 風格的身心追蹤 prototype，目前已有睡眠、情緒、飲食等維度的紀錄功能。睡眠紀錄採用三步驟引導式流程（感受 → 時長 → 總結），使用者反應良好。運動是 BottomBar 五大維度之一（今天 / 睡眠 / 運動 / 飲食 / 情緒），目前尚未實作內容。

### 參考模式：睡眠紀錄

| 項目 | 睡眠做法 |
|------|---------|
| 首頁結構 | 今日摘要卡 + 本週趨勢圖 + insight 建議卡 |
| 紀錄流程 | Step1 感受 → Step2 時長 → Step3 總結 |
| 資料維度 | 感受（5 級）+ 時長（小時）|
| 頻率 | 一天一筆 |
| State 欄位 | `sleepFeel`, `sleepHours`, `sleepRecorded`, `sleepWeekly` |

---

## 問題

使用者缺乏一個輕量、低壓力的方式來記錄每日運動。市面上的運動 app 偏向「數據導向」（卡路里、心率、配速），對於只想簡單回顧「今天有沒有動一動、感覺如何」的人來說太重了。

Pace 的定位是**身心覺察**而非運動績效追蹤，運動紀錄應保持同樣的溫暖、低門檻調性。

---

## 目標使用者

重視身心平衡、希望養成運動習慣但不想被數據壓力綁住的人。他們可能：
- 運動頻率不固定，有時散步、有時瑜伽
- 不在意精確的卡路里或心率
- 想要的是「有記錄到」的成就感和「回顧本週」的滿足感

---

## 成功指標

> 因為是 prototype，以下為設計驗證層面的指標：

| 指標 | 目標 | 驗證方式 |
|------|------|---------|
| 紀錄完成率 | 進入 flow 後 >80% 完成到最後一步 | Prototype 測試觀察 |
| 操作時間 | 整個紀錄流程 <30 秒 | Prototype 測試計時 |
| 使用者感受 | 「簡單」「沒有壓力」 | 測試後訪談 |

## MVP 方案

### 資料模型

```typescript
// 新增到 PaceState
exerciseType: string;          // 選中的運動類型 key
exerciseDuration: number;      // 時長（分鐘），5 分鐘為單位，範圍 5-120
exerciseIntensity: number;     // 強度感受 0-4（對應 MOOD_SCALE：輕鬆/微喘/中等/吃力/爆汗）
exerciseMood: number;          // 運動後心情 0-4（對應 MOOD_SCALE）
exerciseRecorded: boolean;     // 今日是否已記錄
exerciseWeekly: number[];      // 7 天時長陣列（同 sleepWeekly 結構）
```

### 畫面與流程

運動紀錄嘗試與睡眠不同的互動方式：**單頁卡片式輸入**，所有欄位在同一頁完成，透過垂直滾動依序填寫，減少頁面跳轉的認知負擔。

#### 1. ExerciseHome（運動首頁）

結構與 SleepHome 對齊：

```
┌─────────────────────────┐
│  ← 運動                  │  TopBar
├─────────────────────────┤
│                         │
│  ┌─────────────────┐    │
│  │ 🏃 跑步  30 分鐘  │    │  今日摘要卡
│  │ 強度：中等        │    │  （未記錄時顯示 empty state）
│  │ 心情：😊 輕盈     │    │
│  └─────────────────┘    │
│                         │
│  [重新記錄]              │
│                         │
│  本週趨勢                │
│  ┌─ ─ ─ ─ ─ ─ ─ ─ ┐   │
│  │ █   █       █    │   │  7 天長條圖
│  │ █ █ █   █   █    │   │  Y 軸 = 時長（分鐘）
│  │ █ █ █ █ █   █    │   │  今天高亮
│  │ 一二三四五六日     │   │
│  └─ ─ ─ ─ ─ ─ ─ ─ ┘   │
│  本週共 3 次 · 平均 28 分 │
│                         │
│  ┌─────────────────┐    │
│  │ 🌿 小觀察        │    │  Insight 卡
│  │ 這週你持續在動，   │    │
│  │ 身體會記住的。     │    │
│  └─────────────────┘    │
│                         │
│  [今天/睡眠/運動/飲食/情緒]│  BottomBar
└─────────────────────────┘
```

**Insight 邏輯：**

| 條件 | 文案方向 |
|------|---------|
| 本週 ≥3 次 | 正向鼓勵：「這週你持續在動，身體會記住的。」 |
| 本週 1-2 次 | 溫和提醒：「動一動就很好了，不用多。」 |
| 本週 0 次 | 輕柔邀請：「找一個你喜歡的方式，動十分鐘就好。」 |

**週趨勢指標：** Y 軸使用**時長（分鐘）**，同時在圖表下方顯示「本週共 N 次」作為頻率輔助資訊。選擇時長而非次數，是因為時長能反映投入程度的差異（散步 15 分 vs 重訓 60 分），比單純打勾更有資訊量。

#### 2. ExerciseRecord（單頁卡片式記錄）

與睡眠的 step-by-step 不同，採用**單頁垂直流**：使用者在同一頁面內依序完成所有輸入，每個區塊像一張卡片，滾動即可填完。好處是心理上感覺「只有一步」，且可以隨時回頭修改。

```
┌─────────────────────────┐
│  ✕            記錄運動    │  TopBar（關閉按鈕）
├─────────────────────────┤
│                         │
│  今天做了什麼？           │  Section 1: 運動類型
│                         │
│  ┌────┐ ┌────┐ ┌────┐  │
│  │ 🚶 │ │ 🏃 │ │ 🧘 │  │  Grid 選擇（2-3 列）
│  │散步 │ │跑步 │ │瑜伽 │  │  選中項高亮 + 微放大
│  └────┘ └────┘ └────┘  │
│  ┌────┐ ┌────┐ ┌────┐  │
│  │ 🏋️ │ │ 🚴 │ │ 🏊 │  │
│  │重訓 │ │騎車 │ │游泳 │  │
│  └────┘ └────┘ └────┘  │
│  ┌────┐ ┌────┐ ┌────┐  │
│  │ ⛰️ │ │ 💃 │ │ 🤸 │  │
│  │爬山 │ │舞蹈 │ │伸展 │  │
│  └────┘ └────┘ └────┘  │
│                         │
│  ── ── ── ── ── ── ──  │  分隔線
│                         │
│  動了多久？               │  Section 2: 時長
│                         │
│        30 分鐘            │  大字顯示
│  ●━━━━━━━━━━━━━○        │  滑桿（5-120 分鐘，5 分鐘步進）
│  5              120      │
│                         │
│  ── ── ── ── ── ──  ──  │
│                         │
│  強度感覺？               │  Section 3: 強度
│                         │
│  輕鬆  微喘  中等  吃力  爆汗│  5 級橫向選擇
│        ●                 │  （複用 MoodSlider 樣式）
│                         │
│  ── ── ── ── ── ── ──  │
│                         │
│  運動完的心情？            │  Section 4: 心情
│                         │
│  😫  😐  🙂  😊  🥰    │  5 級表情選擇
│              ●           │  （複用 MoodSlider）
│                         │
│                         │
│  ┌─────────────────┐    │
│  │    記錄完成 ✓     │    │  完成按鈕
│  └─────────────────┘    │
│                         │
└─────────────────────────┘
```

**互動細節：**
- 運動類型 grid：點擊選擇，選中項以 `theme.terracotta` 邊框 + 背景色高亮，微放大 `scale(1.05)` 動畫
- 時長滑桿：複用 `HoursSlider` 元件模式，改為分鐘制（5-120，步進 5）
- 強度 & 心情：複用 `MoodSlider` 元件，只改 label 文字
- 完成按鈕：所有欄位都選擇後才啟用（類型 + 時長 + 強度為必填，心情可選填）
- 點擊完成後：設定 `exerciseRecorded = true`，短暫顯示 ✓ 動畫，自動返回 ExerciseHome

**為什麼不用 step-by-step？**
- 運動紀錄的欄位之間獨立性高（類型、時長、強度沒有前後依賴），不像睡眠的「先感受再量化」有敘事意義
- 單頁呈現讓使用者一眼看到「要填什麼」，心理預期更明確
- 可隨時上下滾動修改，不用「回到上一步」

### 導航結構

```
HomeScreen
  └─ BottomBar「運動」tab
       └─ ExerciseHome
            ├─ [開始記錄] / [重新記錄] → push('exerciseRecord')
            │    └─ [記錄完成] → replace('exerciseHome')
            └─ BottomBar 切換其他 tab
```

## 風險評估

| 風險 | 類型 | 影響 | 緩解方式 |
|------|------|------|---------|
| 單頁式記錄在小螢幕上太擁擠 | U（易用性）| M | 實作後在 380×820 frame 內測試，必要時改回 step 模式 |
| 10 種運動選項太多，選擇困難 | U（易用性）| L | 先上線觀察，必要時縮減到 6 種或加入「最近使用」排序 |
| 與睡眠功能 UI 太像，缺乏新鮮感 | V（價值）| L | ExerciseHome 結構對齊但記錄流程刻意差異化（單頁 vs 步驟） |
| 強度量表不直覺 | U（易用性）| M | Label 用體感描述（輕鬆/微喘/中等/吃力/爆汗）而非數字 |

## Open Questions

| 問題 | 假設 | 驗證方式 |
|------|------|---------|
| 使用者偏好單頁式還是步驟式？ | 單頁更快更直覺 | Prototype A/B 測試 |
| 10 種預設運動是否足夠？ | 涵蓋 90% 日常運動 | 使用者訪談 |

## 技術備註

- 導航：使用 NavStack 的 `push` / `replace`，不需 router
- State：直接 mutate `PaceState` 物件，prototype 寫法
- 樣式：inline style + design tokens，不引入 CSS framework
- 元件複用：`MoodSlider`（強度 & 心情）、`HoursSlider` 改造版（時長）、`BlobShape`（裝飾）、`AnimatedEnter`（進場動畫）
- i18n：所有 user-facing 文字走 `theme.L`，需同時新增 zh-TW / en / ja
