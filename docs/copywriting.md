# Pace — Copywriting Guideline

寫給 Pace 這個產品的文案準則。所有 user-facing 的字都走這份規範。

## 一、核心語氣（Voice）

Pace 是一個「陪你看見自己」的產品，不是教練、不是醫療儀器、不是 gamification app。文案要像**一個溫柔而自在的朋友**，不催促、不評價、不擺姿態。

三個必守的原則：

1. **陪伴 > 指導** — 不告訴使用者該做什麼，而是幫他看見自己。
2. **觀察 > 判斷** — 用「我們注意到…」而不是「你應該…」。
3. **溫柔 > 鼓勵** — 承認疲憊、允許跳過，比喊加油更有力量。

## 二、語氣（Tone）分場景

| 場景 | 語氣 | 範例 |
|---|---|---|
| 問候 | 親近、直呼其名 | 「早安，Anniie」 |
| 引導輸入 | 提問、給台階 | 「昨晚，大概睡了多久？」／副標「不用精準，你記得的就好。」 |
| 完成回饋 | 肯定努力，不誇大 | 「今天的你，很棒」「身體會謝謝你」 |
| 觀察 / Insight | 觀察語氣、主動 disclaim | 「這不是規則，只是一個觀察。」 |
| 空狀態 | 不指責缺漏 | 「還沒記錄」而不是「未完成 / No data」 |
| 錯誤 | 盡量不寫錯誤訊息（prototype 層級），若必要用中性口吻 | — |

## 三、文字結構模式

### 主標 + 軟化副標（最常用的雙層結構）

主標**問一件事**，副標**給台階**，讓使用者不怕答錯。

```
你今天醒來，
感覺如何？
—— 先聽聽身體，數字等等再說。
```

```
昨晚，
大概睡了多久？
—— 不用精準，你記得的就好。
```

```
剛剛吃了什麼？
—— 大概的感覺就好。
```

**結構公式**：主標是開放問句或邀請 → 副標是「不用 / 大概 / 隨便 / 也沒關係」的允許句。

### 換行即節奏

標題內用 `\n` 斷句製造呼吸感，通常斷在語氣轉折處：

```
你今天醒來，\n感覺如何？
動了，\n身體謝謝你。
起伏一點，\n整體算是平穩的。
```

一個螢幕上的主標不應該超過兩行。

## 四、用字規則

### ✅ 採用

- **第二人稱「你」** — 直接、親近；避免「使用者 / 用戶」
- **「還沒」** — 不是「未 / 沒有」（如：還沒記錄、還沒動）
- **柔動詞** — 記下、看看、聽聽、留意、注意到
- **情感形容詞作為量尺** — 清淡 / 均衡 / 放縱，輕鬆 / 微喘 / 中等 / 吃力 / 爆汗，疲憊 / 平淡 / 還行 / 輕盈 / 充滿能量
- **小字眼** — 「小觀察」而不是 "Insight"；「這週的你」而不是 "Weekly Report"
- **允許句** — 「跳過也沒關係」、「不用多」、「你記得的就好」

### ❌ 避免

- **數值化的壓力詞** — 目標、達標、完成率、連勝、挑戰、打卡
- **道德 / 表揚詞** — 加油、堅持、棒棒、你最棒
- **系統語言** — 儲存成功、提交、錯誤、失敗、必填、請輸入
- **誇張標點** — 驚嘆號 `!`、全形驚嘆號 `！`（toast 也不要）
- **英文技術詞** — Submit / Save / Cancel / Error，走中文柔化版
- **醫療口吻** — 建議、警示、異常、不足（寧可用「少了點」、「可以多一點」）

## 五、按鈕文案（Buttons / CTA）

| 類型 | 用字 | 不要用 |
|---|---|---|
| 前進 | 繼續、看看昨晚、開始記錄 | 下一步、Next |
| 儲存 | 記下、完成 | 儲存、提交、Save |
| 取消 / 跳過 | 跳過也沒關係、好的，謝謝 | 取消、Cancel、略過 |
| 返回 | 回首頁 | 返回、Back |
| 重做 | 重新記錄 | 重設、Reset |

按鈕文字**不加標點**，但帶情緒的 CTA 可以保留語氣（例：「好的，謝謝」）。

## 六、空狀態與引導

空狀態要**把缺漏變成邀請**，不是空白提醒。

```
還沒記錄今天的睡眠
→ 開始記錄
```

```
還沒動
→ 動一動就很好了，不用多。
```

絕對不要「No data available」、「Nothing to show」、「記錄為空」這種句型。

## 七、Insight / 觀察文案

Insight 永遠要：

1. **主觀柔化** — 「好像」「似乎」「可能」「有點」
2. **自我 disclaim** — 「這不是規則，只是一個觀察」
3. **留餘地給使用者判斷** — 「如果你覺得有幫助，可以留意看看」

範例：

> 有運動的那幾天，你的睡眠好像比較穩。
> 這不是規則，只是一個觀察。如果你覺得有幫助，可以留意看看。

避免絕對化語氣：「你的睡眠取決於運動」、「請多運動以改善睡眠」。

## 八、Toast / 即時回饋

- **一句話** — 最多一行
- **肯定當下，不預設明天** — 「今天的你，很棒」勝過「繼續保持！」
- **不用感嘆號**
- **不強調數字** — 避免「連續 3 天！」

## 九、標點與排版規則

| 情境 | 寫法 |
|---|---|
| 引言 | 用「」，不用 "" 或 '' — 例：「照顧自己，不是一件需要打勾的事。」 |
| 破折號 | 用 `—`（em dash），不是 `--` 或 `-` |
| 省略號 | 用 `…`，不是 `...` |
| 分隔符 | 用 `·`（中點）— 例：「四月 · 台北 · 雨後」 |
| 中英混排 | 中文與英文 / 數字之間加**半形空格** — 例：「4 月 12 – 18」、「每天 22:30」 |
| 句末 | 陳述句用句號「。」，標題、按鈕、label 不加 |
| 範圍 | 用 `–`（en dash）— 例：「4 月 12 – 18」 |

## 十、多語系對照

所有文案進 [src/data/i18n.ts](../src/data/i18n.ts) 的 `PACE_I18N`，**三個語系（zh-TW / en / ja）同步加**。翻譯時保留同等的「留白感」，不要字面直譯：

- `zh-TW`：「不用精準，你記得的就好。」
- `en`：「No need to be exact — just what you remember.」
- `ja`：「正確でなくていい、覚えている範囲で。」

三個語系在**情緒強度**上要一致，不要某一語系變得更商務或更輕浮。

## 十一、快速對照範例

| ❌ 不要這樣寫 | ✅ 改這樣 |
|---|---|
| 請輸入您的睡眠時數 | 昨晚，大概睡了多久？ |
| 儲存成功！ | 完成 |
| 目標達成，恭喜！ | 今天的你，很棒 |
| 資料不足，無法分析 | 還沒記錄 |
| 建議您多運動以改善睡眠 | 有運動的那幾天，你的睡眠好像比較穩 |
| 加油！繼續堅持！ | 你已經連續三天留意自己了，這不容易。 |
| 取消 | 跳過也沒關係 |
| 連勝 5 天 🔥 | 連續紀錄 · 5 天 |

## 十二、Checklist — 寫完一段文案前問自己

- [ ] 有沒有把使用者當朋友、不是病人或學生？
- [ ] 主標有沒有搭配「允許他答得不準」的副標？
- [ ] 有沒有不小心用了表揚、感嘆號、目標達成語氣？
- [ ] 換行節奏自然嗎？念出來順嗎？
- [ ] 三個語系都加了、情緒強度對齊了嗎？
- [ ] 標點用對了嗎（「」／ — ／ · ／ 半形空格）？
